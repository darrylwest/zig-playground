REST API Endpoints

This server provides a simple REST API with the following endpoints. All endpoints use HTTP GET method and return either plain text or JSON responses.

Health Check

URL: /api/v1/health

Method: GET

Response Type: text/plain

Description: Simple health check endpoint that returns "ok" when the server is running.

Example Response:

ok

Try it now →

Ping Test

URL: /api/v1/ping

Method: GET

Response Type: text/plain

Description: Connectivity test endpoint that responds with "PONG".

Example Response:

PONG

Try it now →

Version Information

URL: /api/v1/version

Method: GET

Response Type: text/plain

Description: Returns the current server version in semantic versioning format.

Example Response:

0.1.0

Try it now →

Server Status

URL: /api/v1/status

Method: GET

Response Type: application/json

Description: Returns detailed server status information including uptime, session count, and current timestamp.

Example Response:

{
  "uptime": "3600",
  "session_count": 42,
  "timestamp": "2025-01-15T10:30:45Z"
}

Try it now →

Response Format

  • Success: HTTP 200 OK with appropriate content type
  • Not Found: HTTP 404 for invalid endpoints
  • Method Not Allowed: HTTP 405 for unsupported HTTP methods
  • Bad Request: HTTP 400 for malformed requests

Usage Notes

  • All timestamps are in ISO 8601 format (UTC)
  • Uptime is measured in seconds since server start
  • Session count tracks total connections since server start
  • No authentication required for any endpoints
  • CORS headers are not currently implemented
